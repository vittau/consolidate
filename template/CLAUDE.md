## Skill graph

- `new-entry` → invokes `consolidate` first, then creates today's daily file.
- `consolidate` → autonomous; writes to `entries/weekly/<YYYY-MM>/` and `entries/monthly/<YYYY>/`, appends keyword blocks to entries in `entries/daily/<YYYY-MM>/`.
- `when`, `explain` → read-only queries over the tree.

If the user's request matches one of these skills but they didn't explicitly invoke it (via `/<skill>`), confirm before acting — suggest the appropriate skill and wait for approval instead of running it manually.

## File layout

Entries are grouped into subdirectories to keep each level manageable:

```
entries/
  daily/    <YYYY-MM>/<YYYY-MM-DD>.md   ← daily, grouped by the year-month of the date
  weekly/   <YYYY-MM>/<YYYY-Www>.md     ← weekly, grouped by the month of the ISO week's Thursday
  monthly/  <YYYY>/<YYYY-MM>.md         ← monthly, grouped by year
```

The week's folder rule (month of the Thursday) is the same one `consolidate` uses to decide which monthly summary a week belongs to — so a weekly file always ends up in the same subdirectory as the month it belongs to. Skills that grep must use `*/*.md` at each layer to cover all files.

## Conventions

- ISO 8601 for all dates. ISO weeks (Mon–Sun), labeled `YYYY-Www` with a zero-padded week number.
- Keywords are lowercase, hyphen-separated when multi-word (`performance-review`, not `Performance Review`).
- The keyword block goes at the **end** of every entry, preceded by a `---` delimiter and a single blank line:

  ```
  …user content…

  ---
  keywords: [word1, word2, word3]
  ```

- The `keywords: [...]` line **always stays on a single line**, even if it exceeds prettier's `printWidth` (hence `proseWrap: preserve`). The `when` and `explain` skills grep for `^keywords:` and rely on a single match per file.

- "Complete" periods:
  - Week — complete as soon as its Sunday is strictly before today.
  - Month — complete as soon as the first day of the next month is ≤ today.

## Keyword extraction rules

- Daily: 5–15 keywords. Prioritize concrete nouns — people, projects, places, events, recurring themes.
- Weekly: 10–20 keywords. Most relevant union of the daily keywords for that week.
- Monthly: 15–30 keywords. Most relevant union of the weekly keywords for that month.
- Deduplicate. No generic filler (`work`, `stuff`, `misc`).

## Query skills

- Walk top-down: monthly → weekly → daily. Go straight to daily only when the upper layers haven't been built yet for that period.
- Use `grep` against `^keywords:` lines for stricter matches, and a case-insensitive grep over the full file as a fallback.
- Synonyms are generated by the model from the query (3–8 for `explain`, 2–5 for `when`). No dictionary file.
- Always cite specific dates (`YYYY-MM-DD`) when reporting findings.

## Tooling

- [.claude/skills/consolidate/detect.js](.claude/skills/consolidate/detect.js) prints pending consolidation work. It's the first step of `consolidate`. Pure Node.js (≥18), no dependencies. Also available as `npm run detect`.
- [package.json](package.json) exists to mark the project as ESM (`"type": "module"`) and expose the `detect` npm script. No dependencies.
- No build, lint, or test commands — this is a content repository.

## What not to do

- Don't edit user-written content in a daily entry. Only append the keyword block.
- Don't regenerate an existing weekly or monthly file unless the user explicitly asks.
- Don't add sections (mood, gratitude, reflection) to the daily template. Keep it minimal — structure is the user's choice.
- Don't use the BuJo `>` migration symbol; consolidation handles carry-forward.
