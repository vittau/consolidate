# {{projectName}} ⚡️

Personal journaling framework built on Claude Code skills. You write daily entries in a simplified Bullet Journal style; the framework indexes them incrementally, consolidates them into weekly and monthly summaries, and makes everything searchable by date and by subject.

## Skills

| Skill | Argument | What it does |
|---|---|---|
| `/new-entry` | none | Runs `/consolidate` first, then creates today's daily file if it doesn't already exist. |
| `/consolidate` | none | Auto-detects pending work: adds keywords to daily entries, summarizes complete ISO weeks into weekly files, and summarizes complete months into monthly files. |
| `/when <query>` | text | Finds when an event / person / topic appears. Walks monthly → weekly → daily to narrow the period. |
| `/explain <subject>` | text | Synthesizes what the journal says about a subject. Uses keyword matching with synonym awareness from the model. |

## Directory structure

Entries are grouped into subdirectories to avoid huge single-layer listings:

```
entries/
  daily/    <YYYY-MM>/<YYYY-MM-DD>.md   ← written by the user, grouped by year-month
  weekly/   <YYYY-MM>/<YYYY-Www>.md     ← generated by /consolidate, grouped by the month of the ISO week's Thursday
  monthly/  <YYYY>/<YYYY-MM>.md         ← generated by /consolidate, grouped by year
```

The year-month of the weekly file follows the ISO convention: a week "belongs" to the month that contains its Thursday.

## Bullet Journal notation

Every entry is a standard markdown list item. The BuJo symbol goes inside parentheses
right after the list marker (all ASCII, single-keystroke on a US International keyboard):

- `- (.)` task
- `- (x)` completed task
- `- (<)` scheduled task
- `- (o)` event
- `- (-)` note

Parentheses are used instead of brackets because `- [x]` and `- [ ]` collide with
GitHub-Flavored Markdown task list syntax (they render as clickable checkboxes).

Example:

```markdown
# 2026-04-16 · Thursday

- (.) draft Q2 roadmap
- (x) reply to Marina
- (<) dentist 2026-04-22
- (o) team offsite kickoff
- (-) bike tire needs replacing
```

The classic `>` migration symbol is intentionally omitted — `/consolidate`
carries unfinished items forward automatically.

## Entry format

Daily entries are free-form markdown. Sections, headings, and structure are entirely up to you.

When `/consolidate` processes a daily entry, it appends a keyword block to the end:

```
---
keywords: [meeting, project-x, performance-review]
```

Weekly and monthly files use the same block, placed after a narrative summary.

## How consolidation works

- A week is **complete** as soon as its Sunday is strictly before today.
- A month is **complete** as soon as the first day of the next month is on or before today.
- `/consolidate` processes only what's missing. It never edits user-written content in a daily entry — it only appends the keyword block.
- Keyword search (in `/when` and `/explain`) compares against the stored keyword blocks; synonyms are resolved by the model, with no external dictionary.

## Before you start

Edit [ABOUT.md](ABOUT.md) with a short description of yourself — the file is auto-loaded in every Claude session and gives the skills context to better interpret your entries.

## Typical day

1. `/new-entry` — `/consolidate` runs first to process any pending indexing; then today's file is created or shown.
2. Jot entries throughout the day in BuJo notation.
3. Tomorrow, the cycle repeats. Over time, the weekly and monthly layers accumulate automatically.
4. Ask questions with `/when` and `/explain`.
